Verifiable Agent Trust Envelope Reference Architecture

A reference view of this draft as a portable trust / permit / receipt layer spanning controller, principal, runtime, permit, status, and receipt semantics while composing with existing transport and identity systems.

Draft Boundary

  • - This draft does not replace A2A, MCP, OAuth, VC, or workload identity.
  • - It adds a portable trust envelope across controller, principal, actor, runtime, permit, status, and receipt.
  • - The holder collects portable artifacts before an external action leaves the local surface.

Verifier-Centered Flow

  • - The practical wedge in this repo is verifier centered, especially for AL2 external write decisions.
  • - Recommended order is status, credential, runtime, permit, then local policy.
  • - A verifier can allow, attenuate, or deny after the chain is consistent.

Core Objects

  • - APC anchors actor identity and assurance.
  • - ARP proves the current runtime is fresh and bound to the actor.
  • - AMP scopes what may happen; ASN can narrow it; AER records what actually happened.
