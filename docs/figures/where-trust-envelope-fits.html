Where Verifiable Agent Trust Envelope Fits

This draft composes with discovery, authorization, credential, and runtime layers while adding portable trust / permit / receipt semantics for external agent actions across vendors and surfaces.

What This Draft Adds

  • - actor / principal / runtime separation
  • - portable permit / receipt / status semantics

What This Draft Does Not Replace

  • - A2A, MCP, OAuth, VC, and workload identity stay in place
  • - This draft composes with them instead of re-specifying them

Why It Matters

  • - external agent actions need portable trust context across vendors and surfaces
  • - verifiers need more than discovery and raw credentials alone
