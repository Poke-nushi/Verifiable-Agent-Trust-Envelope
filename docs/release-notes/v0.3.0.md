# VATE v0.3.0 Release Notes Draft
## AL2 evidence reference hardening

This file describes the intended `v0.3.0` discussion-draft pre-release.
`v0.3.0` supersedes the archived `v0.2.0` review snapshot for current
main-branch review purposes. It does not mutate the `v0.2.0` GitHub release or
Zenodo archive artifacts.

## Summary

`v0.3.0` keeps the AL2 verifier admission boundary from `v0.2.0` and hardens
admission request evidence reference handling.

The main machine-readable change is that AL2 admission requests now require at
least one digest-addressed `evidence_refs` entry. An empty `evidence_refs` array
is schema-invalid and should fail closed with `SCHEMA_INVALID` and
`FAIL_CLOSED`.

## Changed

- current AL2 profile identifiers are updated to
  `VATE-AL2-Verifier-Admission-v0.3`;
- current admission, receipt, metadata, report, and SUT result schemas validate
  the v0.3 profile boundary;
- the reference verifier and conformance runner now operate against the v0.3
  corpus by default;
- `docs/release-notes/v0.2.0.md`, `docs/release-gate-v0.2.0.md`, and
  `CITATION.cff` now describe the archived May 5, 2026 `v0.2.0` snapshot
  rather than current hardening work;
- the A2A-shaped metadata review wording remains narrow and does not claim an
  official A2A extension, endorsement, certification, SDK, middleware package,
  or general compatibility proof.

The v0.3.0 hardening release carries forward the A2A-shaped metadata review
package through v0.3-named documents while keeping the same non-official,
metadata-only claim boundary.

## What Implementers Can Test

The draft conformance corpus gives named expected outcomes for 63 AL2 v0.3
cases. Implementers can run their verifier against `corpus.json`, emit a SUT
result file matching `schemas/sut-result.schema.json`, and compare it with the
reference runner.

Minimal comparison command:

```bash
python3 scripts/vate_conformance.py compare \
  --corpus-root conformance/al2-vate-v0.3 \
  --sut-results examples/conformance/sut-results-pass.example.json \
  --report /tmp/vate-v0.3-compare-report.json \
  --implementation-report /tmp/vate-v0.3-implementation-report.json
```

Passing this comparison means one SUT result matched one corpus snapshot. It
does not grant a compatibility badge, certification, endorsement, or production
approval.

## Not Production Ready

This release does not provide:

- final proof packaging requirements;
- a multi-implementation certification suite;
- a normative A2A extension accepted by A2A governance;
- production JOSE / PKI signature verification;
- production security review;
- stable persistent namespace assignment.
